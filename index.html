Hasil Segmentasi Jalan Dengan Deep Leaning U-Net

Metode

Esktraksi Segmentasi Jalan Dengan Deep Learning U-Net

Metode U-Net merupakan model image segmentation yang dasarnya merupakan fully convolutional network tanpa adanya dense layer. Dinamakan U-Net karena bentuk pemodelannya yang menyerupai huruf "U". Data pengolahan yang digunakan yaitu Data Foto Udara Sentul sebanyak 2 NLP. Pengolahan dengan U-Net dilakukan dengan Google Colab, didapatkan pembentukan model dan hasil prediksi. Hasil prediksi pengolahan data disajikan dengan visualisasi pada Web Map.

Procedure

Penjelasan dan Langkah Pengolahan Segmentasi Jalan

1. Data Preparation

Data foto udara yang digunakan memiliki resolusi dan kualitas foto yang baik. Data tersebut dilakukan pre-processing terlebih dahulu dengan menentukan training data dan testing data pada tiap piksel. Visualisasi digitasi jalan yang digunakan sebagai ground truth ditampilkan menggunakan mask layer dengan sajian tiap tiles.

2. Generate Images and Mask Tiles

Data foto udara atau images beserta mask layer dalam bentuk tiles yang memiliki ukuran lebih kecil digunakan dalam proses training model deep learning. Menggunakan modul GeoTile, dilakukan generate tiles pada data training dan data testing yang akan dibentuk menjadi format data raster melalui rasterization. Memanfaatkan modul Numpy untuk menyimpan data tiles pada bentuk file numpy array.

3. Augmentation Data

Pendefinisian training data dan testing data dalam format *.npy yang sudah dilakukan akan dilakukan pembentukan data augmentasi pada setiap images dengan mask. Nilai akan disimpan dalam bentuk list yang kemudian dikonversi pada numpy. Hasil tersebut akan digabungkan dengan citra asli sehingga terbentuk citra gabungan dari hasil augmentasi dan data citra asli.

4. U-Net Model Development

Membangun model segmentasi U-Net dengan pembentukan encoder yang bertujuan untuk mengekstrak fitur penting dari input melalui serangkaian konvolusi dan pooling. Kemudian menggunakan decoder untuk mengembalikan gambar pada resolusi aslinya dengan upsampling. Fitur yang telah diekstraksi digabungkan kembali dengan encoder. Proses paling penting yaitu training model menggunakan tensorflow dilakukan dengan mengatur beberapa parameter seperti epoch training, batch size, dan patience.

5. Evaluasi Model Loss dan Membentuk Prediksi

Kualitas model yang terbentuk setelah proses training divisualisasikan dengan nilai data loss yang diperoleh dari data train dan data testing. Prediksi dilakukan menggunakan testing data dengan model yang telah dihasilkan. Hasil prediksi berpengaruh terhadap nilai threshold yang digunakan sebab berkaitan dengan ambang batas probabilitas.

6. Accuracy Assessment

Evaluasi model klasifikasi citra menggunakan accuracy assessment diterapkan pada beberapa tahap, salah satunya validation set. Proses ini dilakukan untuk turning hyperparameter serta pemilihan model terbaik. Akurasi hasil prediksi didapatkan dengan perhitungan nilai True Positives, True Negatives, False Positives, dan False Negatives sebagai modal dalam melakukan perhitungan accuracy assessment. Nilai ini digunakan untuk menghitung nilai Precision, Recall, F1 Score, dan Intersection of UnionÂ (IoU).
